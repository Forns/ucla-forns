Search Problems

The following tree represents some imaginary search tree composed of lettered states, costs along edges, and evaluation function values at nodes in the form (g(n), h(n)), where g(n) is the cost it took to reach that node, and h(n) is some admissible heuristic's score of that node to some goal state. Goal states (F, O) are listed in green.



Assuming a left-to-right node expansion and generation order (and not stopping at goal states), in what order will uninformed Depth First Search traverse this tree?

A, B, E, F, C, G, L, H, M, N, I, D, J, K, O


If our uninformed DFS does stop at a goal state, which will it find? Is this an optimal goal? Is DFS guaranteed to give us an optimal goal if one exists?

Uninformed DFS stops at goal state F, which happens to be the optimum goal in this tree, but in general, DFS is not an optimal search strategy.


Assuming a left-to-right node expansion and generation order (and not stopping at goal states), in what order will uninformed Breadth First Search traverse this tree?

A, B, C, D, E, F, G, H, I, J, K, L, M, N, O


If our uninformed BFS does stop at a goal state, which will it find? Is this an optimal goal? Is BFS guaranteed to give us an optimal goal if one exists?

Uninformed BFS stops at goal state F, which happens to be the optimum goal in this tree, and in general, BFS will give us the optimal goal for uniform cost problems.


Assuming nodes of equivalent score generated earlier in the process will be expanded first, and that our search does not stop at goal states, in what order will Greedy Search traverse this tree (assuming an admissible heuristic)?

A, D, B, F, C, K, O, J, E, G, H, N, L, M, I


If our greedy search does stop at a goal state, which will it find? Is this an optimal goal? Is greedy search guaranteed to give us an optimal goal if one exists?

Greedy search stops at goal state F, which happens to be the optimum goal in this tree, and in general, if the heuristic is consistent and admissible, will give us an optimal goal for uniform cost problems.



Assuming nodes of equivalent score generated earlier in the process will be expanded first, and that our search does not stop at goal states, in what order will A* Search traverse this tree (assuming an admissible heuristic)?

A, D, B, C, F, K, O, J, E, G, H, N, I, L, M


If our A* search does stop at a goal state, which will it find? Is this an optimal goal? Is A* search guaranteed to give us an optimal goal if one exists?

A* search stops at goal state F, which happens to be the optimum goal in this tree, and in general, if the heuristic is consistent and admissible, will give us an optimal goal in either uniform or non-uniform cost problems.


Alright, let's switch up our search tree so that it's no longer uniform cost!



If our greedy search does stop at a goal state, which will it find? Is this an optimal goal?

Greedy search stops at goal state F, which is NOT optimal! Changing the costs of the problem did not affect its search order.


If our A* search does stop at a goal state, which will it find? Is this an optimal goal?

A* search stops at goal state O, which is the optimum. Changing the costs of the problem did indeed affect its search order.

Constraint Satisfaction Problems

Given N variables each with a domain of V values, what is the time complexity needed in order to find a solution?

O(V^N)


Given the following CSPs, determine whether or not the domains are arc consistent. If they are not, correct them to be arc consistent.

  X = { A, B }
  D = { {1, 2, 3},
        {1, 2, 3, 4, 5, 6, 7, 8, 9} }
  C = { B = A^2 }

Click for answer.

No! The domains of A and B are NOT arc consistent. B has elements in its domain that cannot be valid from any choice for A.
The fix is to set the domain of B = {1, 4, 9} 


  X = { S, T, V }
  D = { {1, 2, 3},
        {1, 2, 3},
        {1, 2, 3} }
  C = { 
        S != T,
        T != V,
        V != S
      }

Click for answer.

Yes! Arc consistent.


  X = { S, T, V }
  D = { {1, 2, 3},
        {1, 2, 3},
        {1, 2, 3} }
  C = { 
        S < T,
        T > V,
        V = S
      }

Click for answer.

No! Not arc consistent. Here are the reasons:

  • S can never be 3 because for all values in the domain of T, T can never be greater than 3. So change domain of S = {1, 2}

  • T can never be 1 because for all values in the domain of V, V can never be less than 1. So change domain of T = {2, 3}

  • V must always be = S, but since we removed 3 from S's domain, we must also remove it from V's as well. So change domain of V = {1, 2}

Determine if the following propositional logic sentences are valid, unsatisfiable, or neither.

  Let S  = Smoke
      Fi = Fire
      H  = Heat
      
  1. (S ⇒ Fi) ⇒ (¬S ⇒ ¬Fi)
  
  2. (S ⇒ Fi) ⇒ ( (S ∨ H) ⇒ Fi)
  
  3. ( (S ∧ H) ⇒ Fi) ⇔ ( (S ⇒ Fi) ∨ (H ⇒ Fi) )

Click for answers TO ALL 3 of the above.

(1) Neither, (2) Neither, (3) Valid


Amongst the 3 sentences in the previous example, which entail one another?

Since we know that sentence 3 is valid, and that neither sentence 1 nor 2 are inconsistent, then both sentence 1 and sentence 2 entail sentence 3! That said, an inspection of the models for sentences 1 and 2 show that neither M(sentence 1) ⊄ M(sentence 2) nor that M(sentence 2) ⊄ M(sentence 1), so neither entails the other.


Perform inference on the following propositional KB to answer the query: α = A ⇒ D


  KB =
     1. (A ∨ ¬B) ⇒ C
     2. C ⇒ (D ∨ ¬E)
     3. E ∨ D
  
  α = A ⇒ D

Click for a sample derivation.

  ; Step 1: Add ¬α to KB:
  ¬α = ¬(¬ A ∨ D)
     = A ∧ ¬D
  
  ; Step 2: Convert KB to CNF
  1. ¬A ∨ C (from 1)
  2. B ∨ C (from 1)
  3. ¬C ∨ D ∨ ¬E (from 2)
  4. E ∨ D (from 3)
  5. A (from ¬α)
  6. ¬D (from ¬α)
  
  ; Step 3: Perform inference
  7. C [1, 5]
  8. E [4, 6]
  9. D ∨ ¬ E [3, 7]
  10. D [8, 9]
  11. ∅ [6, 10]
  → ← Contradiction!
  
  ∴ KB ⊨ α

First Order Logic

Convert the following English sentences into FOL ones using the relation names specified in brackets.

  1. John likes all food. [Food, Likes]
  
  2. Apples are food! [Food]
  
  3. Chicken is food! [Food]
  
  4. Anything that someone eats, that doesn't kill them, is food. [Eats, Kills, Food]
  
  5. For everyone in general, if something kills you, you're not alive. [Kills, Alive]
  
  6. Bill eats peanuts and is very much alive! [Eats, Alive]
  
  7. Anything Bill eats, Sue also eats. [Eats]

Click for solutions to ALL of the above.

  1. ∀x Food(x) ⇒ Likes(John, x)
  
  2. Food(Apples)
  
  3. Food(Chicken)
  
  4. ∀y [∃x Eats(x, y) ∧ ¬Kills(y, x)] ⇒ Food(y)
  
  5. ∀y [∃x Kills(x, y)] ⇒ ¬Alive(y)
  
  6. Eats(Bill, Peanuts) ∧ Alive(Bill)
  
  7. ∀x Eats(Bill, x) ⇒ Eats(Sue, x)

OK, now that we have this riveting story in FOL, let's put it in a usable format.

Convert the above FOL sentences into CNF.

Click for solutions to ALL of the above.

  1. ¬Food(x) ∨ Likes(John, x)
  
  2. Food(Apples)
  
  3. Food(Chicken)
  
  4. ¬Eats(x, y) ∨ Kills(y, x) ∨ Food(y)
  
  5. ¬Kills(x, y) ∨ ¬ Alive(y)
  
  6a. Eats(Bill, Peanuts)
  6b. Alive(Bill)
  
  7. ¬Eats(Bill, x) ∨ Eats(Sue, x)

Treating the above CNF sentences as our knowledge base, prove that KB ⊨ α where α = Likes(John, Peanuts)

Click for a sample derivation.

  ; Step 1: Add ¬α to the KB:
  8. ¬Likes(John, Peanuts)
  
  ; Step 2: KB already in CNF
  
  ; Step 3: Perform inference!
  9. Kills(Peanuts, Bill) ∨ Food(Peanuts) [4, 6a, Θ = { x/Bill, y/Peanuts }]
  
  10. ¬Alive(Bill) ∨ Food(Peanuts) [5, 9, Θ = { x/Peanuts, y/Bill }]
  
  11. Food(Peanuts) [6b, 10, Θ = {}]
  
  12. Likes(John, Peanuts) [1, 11, Θ = { x/Peanuts }]
  
  13. ∅ [8, 12, Θ = {}]
  → ← Contradiction!
  
  ∴ KB ⊨ α
