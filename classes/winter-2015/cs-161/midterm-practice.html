Beam search is a breadth-first-search optimization whereby some subset of nodes along the frontier of each level are expanded according to some heuristic score. The beam width defines the number of best (according to the heuristic) nodes along the frontier that will be expanded with each level.

Suppose our beam width is 2, we use the greedy heuristic in scoring nodes along the frontier (i.e., just h(n)), and in the event of a tie, we will expand the left-most node(s) over the others (nodes along the then limited frontier will be expanded left-to-right). What will be the order of expansion for beam search in the search tree above, assuming we do not stop at goal states?

A, B, D, E, F, J, K, O


In general, is Beam Search complete?

No! What we gain in efficiency from pruning some unfavorable-looking subtrees, we lose in completeness since the subtrees we prune may have contained a solution.

Simulated annealing is a local search strategy where we may make downward or sideways moves with some probability proportional to the amount of time that we've been searching.

Typically, we refer to this probability as a function of our temperature, which (just like a metal cooling) will decrease over time, giving us much exploration of the search space at the beginning until we "settle" upon a local max near the end.

Now, for our purposes, suppose we have the following simulated-annealing pseudocode denoting the probability of moving to some neighbor; we take three parameters as values:

  1. S = current state score (the score of the state we're currently occupying)

  2. N = neighbor score (the score of the neightbor state we're considering)

  3. T = temperature (higher values means we are more likely to explore worse neighbors)


  annealingProbability (S, N, T)
      if (N < S)
          return 1.0
      return exp((S - N) / T)

Suppose our temperature starts out at T = 100, cools at a rate of 10% per action taken, and that we have taken 3 steps to reach state S in the table above. What will be the probability of moving to neighbor N2?

  Here we have taken 3 steps to reach S, meaning our temperature:
  T = (100 * 0.9 ^ 3) = 72.9
  
  Since N > S, this gives us a movement probability of:
  exp((S - N) / T) = exp((5 - 6) / 72.9)
                   = 0.986

...a very high chance to move to N2!


Suppose our temperature starts out at T = 100, cools at a rate of 10% per action taken, and that we have taken 50 steps to reach state S in the table above. What will be the probability of moving to neighbor N1?

  Here we have taken 3 steps to reach S, meaning our temperature:
  T = (100 * 0.9 ^ 50) = 0.52
  
  Since N > S, this gives us a movement probability of:
  exp((S - N) / T) = exp((5 - 7) / 0.52)
                   = 0.021

...a very low chance to move to N1 since we've been exploring for awhile.

Given the following CSPs, determine whether or not the domains are arc consistent. If they are not, correct them to be arc consistent.

  X = { A, B }
  D = { {1, 2, 3},
        {1, 2, 3, 4, 5, 6, 7, 8, 9} }
  C = { B = A^2 }

Click for answer.

No! The domains of A and B are NOT arc consistent. B has elements in its domain that cannot be valid from any choice for A.
The fix is to set the domain of B = {1, 4, 9}

  D = { {1, 2, 3},
        {1, 2, 3},
        {1, 2, 3} }
  C = { 
        S < T,
        T > V,
        V = S
      }

Click for answer.

No! Not arc consistent. Here are the reasons:

  • S can never be 3 because for all values in the domain of T, T can never be greater than 3. So change domain of S = {1, 2}

  • T can never be 1 because for all values in the domain of V, V can never be less than 1. So change domain of T = {2, 3}

  • V must always be = S, but since we removed 3 from S's domain, we must also remove it from V's as well. So change domain of V = {1, 2}

Amongst the 3 sentences in the previous example, which entail one another?

Since we know that sentence 3 is valid, and that neither sentence 1 nor 2 are inconsistent, then both sentence 1 and sentence 2 entail sentence 3! That said, an inspection of the models for sentences 1 and 2 show that neither M(sentence 1) ⊄ M(sentence 2) nor that M(sentence 2) ⊄ M(sentence 1), so neither entails the other.


Perform inference on the following propositional KB to answer the query: α = A ⇒ D


  KB =
     1. (A ∨ ¬B) ⇒ C
     2. C ⇒ (D ∨ ¬E)
     3. E ∨ D
  
  α = A ⇒ D

Click for a sample derivation.

  ; Step 1: Add ¬α to KB:
  ¬α = ¬(¬ A ∨ D)
     = A ∧ ¬D
  
  ; Step 2: Convert KB to CNF
  1. ¬A ∨ C (from 1)
  2. B ∨ C (from 1)
  3. ¬C ∨ D ∨ ¬E (from 2)
  4. E ∨ D (from 3)
  5. A (from ¬α)
  6. ¬D (from ¬α)
  
  ; Step 3: Perform inference
  7. C [1, 5]
  8. E [4, 6]
  9. D ∨ ¬ E [3, 7]
  10. D [8, 9]
  11. ∅ [6, 10]
  → ← Contradiction!
  
  ∴ KB ⊨ α

First Order Logic

Convert the following English sentences into FOL ones using the relation names specified in brackets.

  1. John likes all food. [Food, Likes]
  
  2. Apples are food! [Food]
  
  3. Chicken is food! [Food]
  
  4. Anything that someone eats, that does not kill them, is food. [Eats, Kills, Food]
  
  5. For everyone in general, if something kills you, you are not alive. [Kills, Alive]
  
  6. Bill eats peanuts and is very much alive! [Eats, Alive]
  
  7. Anything Bill eats, Sue also eats. [Eats]

Click for solutions to ALL of the above.

  1. ∀x Food(x) ⇒ Likes(John, x)
  
  2. Food(Apples)
  
  3. Food(Chicken)
  
  4. ∀y [∃x Eats(x, y) ∧ ¬Kills(y, x)] ⇒ Food(y)
  
  5. ∀y [∃x Kills(x, y)] ⇒ ¬Alive(y)
  
  6. Eats(Bill, Peanuts) ∧ Alive(Bill)
  
  7. ∀x Eats(Bill, x) ⇒ Eats(Sue, x)
